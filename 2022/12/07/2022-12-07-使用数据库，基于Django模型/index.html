使用数据库，基于Django模型

发表于2022-12-07|更新于2022-12-11|Django
|阅读量:

Django 模型

  • 在Django框架中，主要通过模型来实现与数据库的交互功能，如数据的增删改查，以及多表关联等
  • 模型用于描述数据库表结构，模型实例可以实现数据操作，一个模型对应一个数据库表，模型中的字段对应数据库表中的一个字段
  • 在定义好模型后，Django会提供数据访问的一整套API，从而自动在数据库中生成相应表，不需要额外使用SQL脚本或者手工创建表，大大提高开发效率

定义模型

  1. 常见模型字段

    模型字段 说明 Mysql数据库对应的字段类型
    AutoField 自增类型 Int
    BooleanField 布尔 Tinyint
    CharField 字符类型 varChar
    DateField 日期字段 Date
    DateTimeField 日期时间字段 DateTime
    IntergerField 整数类型 Int
    TextField 长文本类型 LongText
    TimeField 时间类型 Time
    FloatField 浮点数类型 Double
    FileField 文件类型 varChar
    ImageField 图像类型 varChar
    DecimalField 数值型类型 Decimal(固定精度的十进制数)

  2. 常见字段参数

    字段参数 含义
    verbose_name 设置字段的显示名称
    primary_key 设置字段为主键
    editable 是否可以编辑
    max_length 字段最大长度
    blank 若为True，则允许字段为空字符串，默认为False
    null 若为True，则允许字段为null，默认False
    default 设置字段的默认值
    choices 设置字段的可选值
    db_column 设置表中的列名称，如果不设置，则字段名为列名
    db_index 字段是否可以建立索引
    unique 字段是否可以建立唯一索引
    error_messages 自定义错误信息（字典类型）
    validators 自定义错误验证（列表类型）

  3. __str__方法

    • 方法用来设置模型的返回值，默认返回值为模型对象
    • 可以通过该方法来设置模型的不同返回值
    • 只允许返回字符类型的字段，如果需要返回其他类型，则需要str()进行转换

  4. Meta类

    Meta类是Django模型的一个内部类，用于定义一些Django模型的行为特性，常用参数见下表

    参数 含义
    abstract 若为True，则该模型类为抽象类
    db_table 设置模型对象的数据表名称，若不设置，则名称为‘应用名_模型类名’
    managed 默认为True，管理数据表的生命周期，包括迁移等
    ordering 模型对象返回的记录结果集按照哪个字段排序，一般如下设置：
    Ordering=[“create_date”]按照创建时间升序
    Ordering=[“-create_date”]按照创建时间降序
    Ordering=[“-create_date”，”orderid”]按照创建时间降序，再以订单编号升序
    verbose_name 模型类在后台管理中显示的名称，一般为中文
    index_together 多个字段的联合索引
    unique_together 多个字段的联合约束，比如，购物车表中每一行的用户和商品字段必须唯一，不能重复

了解模型中的关系

​ 数据库中表存在关联关系，而django中的模型对应数据库中的表，因此django中的模型也存在这些关系

  1. “一对一”关系

    ​ 用户基本信息和用户扩展信息就是一个”一对一”关系，在模型中使用OneToOneField()方法来构建模型

    OneToOneField()方法的参数如下：

    • to 要进行关联的模型名称
    • to_field 要进行关联的表中字段名称
    • on_delete 删除关联表中的数据时使用的配置选项(选项如下)
      1. CASCADE 级联删除
      2. PROTECT 采用保护机制抛出错误，不删除扩展表内容
      3. SET_NULL 字段属性null=TRUE时，将关联内容置空
      4. SET_DEFAULT 设置默认值
      5. SET 设置为指定的值
      6. DO_NOTHING 删除基本表时，不做任何操作

  2. “一对多”关系

    ​ 部门信息表和用户信息表是”一对多”关系，一个部门肯定是有多个用户，在模型中使用ForeignKey()方法来构建模型

    ForeignKey()方法的参数如下：

    • to 要进行关联的模型名称
    • to_field 要进行关联的表中字段名称
    • on_delete 删除关联表中的数据时使用的配置选项
    • related_name=None 在反向操作时使用的字段名，用于代替[表名_set]
    • related_query_name=None 在反向操作时使用的连接前缀，用于替换[表名]
    • db_constraint=True 是否在数据库中创建外键约束

  3. “多对多”关系

    ​ 用户扩展信息表和技能信息表是”多对多”关系，一个用户可以有多个技能，一个技能也可以对应多个用户，在模型中使用ManyToManyField()方法来构建模型

    ForeignKey()方法的参数如下：

    • to 要进行关联的模型名称
    • db_constraint=True 是否在数据库中穿件外键约束
    • db_table=None 默认创建的多对多关系表的表明

用Django中的ORM操作数据库

了解ORM

  • ORM在关系型数据库和业务实体对象之间进行映射
  • ORM对底层进行了封装与隔离，在使用Django开发项目时，程序员无须关心程序使用的是mysql还是oracle，在操作数据时也无须和复杂的SQL语句打交道，只需要使用ORM提供的API进行操作即可
  • 每个模型都是一个python类，每个模型都会映射到一个数据库表上
    • 类：数据库中的表
    • 属性：数据库中的字段
    • 实例：数据库表中的一条数据

熟悉QuerySet对象

  • QuerySet对象也称为”查询集”，表示从数据库获取的数据对象集合
  • 查询集有一个最重要的特性——惰性执行，在创建查询集合不会访问数据库，只有在调用相关方法时才会访问数据库
QuerySet对象中常用方法介绍

​ Django的manage工具提供了shell命令，帮助我们直接在终端中执行测试Python语句

  1. all()方法：用于获取模型的QuerySet对象，获取所有的数据

  2. filter()方法：用于实现数据过滤功能，相当于SQL中的where，返回QuerySet对象

  3. get()方法：用于查询数据表记录，以模型对象的形式返回符合要求的数据

  4. exclude()方法：用于排除符合条件的数据，返回QuerySet对象

    如下图：使用exclude(id__lt)来排除id<3的部门，所以查出id=3&id=4的部门

    可以在filter()方法和exclude()方法中使用大小于、模糊匹配等操作符，常见的操作符如下图：

    操作符 含义 具体使用
    __gt 大于 filter(age__gt=20)
    __gte 大于或等于 filter(age__gte=20)
    __lt 小于 filter(age__lt=20)
    __lte 小于或等于 filter(age__lte=20)
    __in 在某个列表内 filter(sex__in=[1,0])
    __contains 模糊匹配 filter(name__contains=’张’)
    __year 日期字段的年份 filter(createdate__year=2021)
    __month 日期字段的月份 filter(createdate__month=12)
    __day 日期字段的天数 filter(createdate__day=31)

  5. Values()方法：用于提取需要返回的字段(类似于SQL: select “id”,”departname” from “app4_departinfo”)，返回QuerySet对象

  6. distinct()方法：用于去除重复数据，返回QuerySet对象

F()函数和Q()函数

  1. F()函数

    ​ 用于实现数据表中字段的各种运算符操作，比如，要给用户扩展表中的多有人员的薪水增加1000元

    使用F()函数，可以直接生成SQL语句进行更新，效率会更高一些

    使用F()函数更新字段后，需要使用refresh_from_db()方法才能获取到最新字段的值

  2. Q()函数

    用于对象进行多条件查询，支持&(and)、|(or)、~(not)操作符

    使用Q()函数查询用户扩展表中年龄大于30岁且薪水大于5000元的人员
